お断り
　windows11にて作製しておりwindows8以下での検証はしていません
　windows11とwindows10での動作を確認しています
　windows11(10)proの方は仮想デスクトップ等で試験されると良いでしょう
　また
　Windowsの仕様、制限によりスタートキーを押すとゲーム画面が最小化してしまう
　フルスクリーンモードではゲーム画面の前に表示することができませんので
　本体の特徴である残時間を利用しての記録ができません
　フルスクリーンでは使えないのではなくゲーム画面と同時に見ることができないので
　消極的な説明としてウィンドウモードか仮想フルスクリーンモード推奨となります
　こちらに関しては後述する入力欄への直接入力で対応して下さい
----------------------------------------------------------------
使い方
　起動しますと巡航船の現状を表示します
　～に航行中停泊中 と [G or R]の文字 と 残時間4桁 が表示されていて
　～に航行中停泊中と残時間4桁のエリアをダブルクリックで作業領域の表示非表示の切替です
　後述するメニューで作業領域表示とあるのも同じ機能です
　　処理の都合によりダブルクリックで開くところと開かないところがあります
　またこのダブルクリックで～に航行中停泊中と表示されている文字がクリップボードに入りますが
　これはwindowsの仕様で避けることができませんのでご理解下さい、これが嫌な方は
　後述するメニューの作業領域表示で切り替えて下さい

[G or R]文字
　左右の文字列に挟まれている[G or R]文字はグレンサーバとレイナサーバの切り替えです

[船 or worldNow]ボタン
船
　巡航船を直接見ている(ゲームをプレイしている)状態で巡航船の次の出航時刻を
　先頭にしてそこから10日分のデータを計算します
worldNow
　ゲーム画面に表示される情報をフォルダ内worldNow.pngを参考に確認、記録します
　worl「dNow=」 が認識できる時に数字を読み取ります
＊system.cfgに1行追加してる方以外無用のものでこれの説明はしません
このボタンは順に船、worldNowと切り替わります
----------------------------------------------------------------
*基本的に7日後に記録し直すものですがメンテのずれ込みがあった場合に備えて
　10日先まで計算しておきますがその10日前の記録で表示しても意味が無いので
　その際はエラーとして「～に航行中停泊中□残時間」を表示しません

[確認]ボタン
　船、worldNowで残時間(worldNowでは経過時間)の情報を本体に認識させるボタンで
　文字と文字が透過している背景によっては認識できないこともありますので
　できるだけ色の変化が少なく文字が背景に紛れないところで確認してください
＊worldNowに関しては測定時の桁を固定できませんので確認ボタンを押して
　「worldNow=桁」で桁が合っているかを確認してから記憶ボタンを押してください
＊船に関しては残1分を切って秒だけの表示になった後は確認できません、分秒がある状態のみの仕様です
----------------------------------------------------------------
フォルダ内に画像を添付していましてそのような感じで覗き窓を動かしてください
以前のverの本体ですが覗き窓の処理は変わっていません
巡航船のアイコンを覗き窓の縁と重なるようにして情報を表示させて
そのまま本体内でタイトルバーにカーソルを移動してドラッグすることで
情報を表示したまま本体を動かせます

[記録]ボタン
　確認して計測日時と対象残時間に記録ができていれば押して本体に記憶させます
＊船で記録する時に船ボタンの右にある選択ボックスで船を見ている港の指定をしてください

計測日時＊
　覗き窓で記録するのとは別に数値を入力も可能です
　数値を入力して 例 20231227102017 必ず14桁にしてください、また月日時分秒は
　2桁で入力してください、2023年1月1日1時1分1秒は20230101010101という書式になります
　入力後[更新]ボタンで反映し、データファイルに保存します

対象残時間＊
　覗き窓で記録するのとは別に数値を入力も可能です
　　worldNowの数値を入れる際は下3桁がミリ秒で4桁以下は反映しません
　　　船の場合は4桁無いと反映しません
　入力後[更新]ボタンで反映し、データファイルに保存します

記録値調整＊
・worldNow
　サーバの[起動]からの経過時間であろうと思いますが
　これを大雑把に増減することで5秒ほど前倒ししておく、遅れた場合に補正する用途にどうぞ
・船
　見た時点で記録を取りますので下の遅延系調整が合っていれば不要ですが
　これを大雑把に増減することで5秒ほど前倒ししておく、遅れた場合に補正する用途にどうぞ
・秒、分それぞれ±59加減できます

遅延系調整＊
　サーバとクライアントの通信に関する様々な遅延に対しての調整となります
　本体計算処理の都合で巡航船の運航ひとつ毎に加算する事で調整します
　値を決めたら[更新]ボタンを押してください。単位はミリ秒で+9.999まで
---------------------------------------------------------------
＊2つの調整は巡航船を見ないと合っているかどうかの判断ができません
(主に)サーバ、クライアント共に処理と通信が安定していれば一度合わせた値で運用できる
のですがサーバの方で不安定な状況が続く場合に金曜までは合っていたが
土日を挟んで月曜に見たらかなりずれていたということもあります
遅延系調整はworldNowと船で共通ですが
記録値調整はそれぞれの記録した時点への増減として独立しています

作業領域の終了
　～に航行中停泊中のエリアや作業領域のボタンや入力枠以外の部分をダブルクリックで非表示になります
　最下部にカラーピッカーもどきでカーソルの位置の色を表示します

PCのマスターボリュームの調整
　本体上にマウスカーソルがある時にマウスホイールの回転で10刻みで増減します
　プログレスバーとRとGの文字のところでPCのマスターボリュームを表示します
　Ctrlを押しながらマウスホイールの回転で1刻みになります
＊ホイールのスクロール量によって変わるかも知れません
----------------------------------------------------------------
本体を右クリックで出るメニュー類
　アプリケーションキーに対しての処理は省いてますので出ても位置がマウスカーソルのところには出ません

グラフィック表示
　数値ではなく簡易なグラフィックにて巡航船の状況を表示します
＊巡航船状況表示というタイトルではこちらが作成意義となる表示です

テキスト表示
　数値で巡航船の状況を表示します、このアプリの標準表示です

作業領域表示
　冒頭で説明したダブルクリックと同じです

コントロールパネル(win)　コントロールパネルをアイコン表示で開きます

エクスプローラ(フォルダ)　を、開きます

画面消灯　ディスプレイのスリープを1分と3分を選んで呼び出します

最小化　タスクバーにアイコンの状態
＊タイトルバーを省略していますのでタスクバーのアイコンをクリックしても
　最小化はできません、本体上で最小化をお選びください、最小化からの復帰はできます

終了
メニューの終了にて、この時点でGかRと荷物か船の組み合わせ
これをモードとして組み合わせて
　Gボタン worldNowボタン で グレンサーバで荷物を記録した → gsd.txt
　Rボタン worldNowボタン で レイナサーバで荷物を記録した → rsd.txt
　Gボタン 船ボタン 　 で グレンサーバで船残を記録した → gcd.txt
　Rボタン 船ボタン 　 で レイナサーバで船残を記録した → rcd.txt
それぞれのファイルをセーブします
またそれぞれのモード時に[記録]と[更新]ボタンを押した時にbackupフォルダにコピーを
作りますので何らかの原因で本体の正常終了が出来なかった時などに
本体フォルダにコピーして起動してみてください
＊本体が起動しなくなった！
　backupフォルダにあるoriginalフォルダのファイルを全部本体のあるフォルダにコピーして
　記録を取り直してください

他の機能
　下で説明します

- - 現在 cfg は A - -
のような表示がありますがcfgを上書きコピーした時に本体としてはA,Bとして記憶しますが
手動でcfgを入れ替えた場合を監視はしていませんので合わなくなりますので
後述するA,Bをbackupフォルダからコピーする際の目安となります

cfg に A を上書き
　backupフォルダに「cfg を A として保存」でコピーした Asystem.cfg を system.cfg に上書きコピーします

cfg に B を上書き
　backupフォルダに「cfg を B として保存」でコピーした Bsystem.cfg をs ystem.cfg に上書きコピーします

他の機能

マウスでウィンドウを最前面
　ドロップダウンメニューにて応答速度を500msから0まで選びます
ON
　マウスの下にあるウィンドウを最前面にします
OFF
　この効果を終了します
＊windowsのコントロールパネルを開いて コンピューターの簡単操作センター を開き マウスを使いやすくします を開きます
その中の項目の マウスポインターをウィンドウに合わせたときにウィンドウを選択します と
似たようなことをしているのでONOFFと応答速度はアプリを終了しても設定は残ります
＊応答速度はこれが実装された時から標準設定として500msをwindowsに埋め込んであるようです

位置固定
　本体の移動、固定の切り替えです、本体終了時に保存はしません

ダークモード、ライトモード
　それっぽく作った白黒表示の切り替えです、起動時にwindowsのモードを見て切り替えるようにはしてありますが
　本体起動中にwindowsの変更は見ていません

本体表示切替
　オリジナルスキン
　　当アプリ固有のスキンです
　スキン無し
　　windows10用の標準表示です、またwindows11ではwindows10と同じ表示になります
　スキン無し(win11)
　　windows11のみ表示されます、windows11専用の角の丸い表示です
この項目は本体終了時に設定を保存します

通常表示(透過終了)
　透過表示に切り替えた後で本体標準の起動直後の表示に戻ります

文字
　色選択 - お馴染みの色選択ウィンドウが出ますのでお好みの色をどうぞ
　フォント  - お馴染みのフォント選択ウィンドウが出ますが何となく付けているだけで
　　　　　　個々のフォントに対して表示調整はしませんのであしからず
　初期設定 - 本体標準のフォントと色に戻します
＊ダイアログを使う経験として入れただけでありまして本体上部のみ反映します

cfg を A として保存 ＊
　system.cfg を Asystem.cfg としてbackupフォルダにコピーします

cfg を B として保存 ＊
　system.cfg を Bsystem.cfg としてbackupフォルダにコピーします

＊基本的に保存はクライアントを終了した時点でAかBに保存してください

　system.cfgそのものはこちらで上書きコピーした場合、そのままになりますのでご注意を

付録 データを保存しているファイル群
mode.txt モードや本体の座標を保存しているファイルです
0    荷物と船の切り替え
100  本体横座標
100  本体縦座標
R    GとRの切り替え
0    4つのデータファイルの内部用番号
　ここから特殊な項目になります
0:20:0.0       巡航船の停泊時間
0:10:21.000  サンライズ半島からツインクラウンまでの航行時間
0:10:47.000  ツインクラウンからサンライズ半島までの航行時間
　航行時間をストップワッチなどで計測して変更することができます
A,B　マルチクライアントでsystemcfgを上書きした時の記憶で初期設定はN
数値　レイナサーバ用の遅延系調整の値です
数値　グレンサーバ用の遅延系調整の値です
数値　本体表示切替のデータです
end ファイルの読み込みの終了用です

rcd.txt gcd.txt 巡航船を見て記録するデータファイルです
20230502135325 巡航船の残時間を記録した時刻です
0000 停泊残時間です
000000 記録値調整用です
0 記録した港のスイッチです 0 ツインクラウン 1 サンライズ半島
end ファイルの読み込みの終了用です

rsd.txt gsd.txt worldNowを見て記録するデータファイルです
＊ゲームエンジンを使ったゲームで解放しているコマンド類なので使えなくなる可能性はあります
20230712211015 worldNowの経過時間を記録した時刻です
11782918 worldNowの値
000000 記録値調整用です
end ファイルの読み込みの終了用です

隠し要素として本体がアクティブでありマウスカーソルが本体上にある時に
Too Many Application というウィンドウを探して、あれば最前面に持ち上げます
書くと隠しにならないという結果は実に悩ましいものです

直接本アプリとは関係ないのですが
systemcfgをアカウントにより上書きコピーで切り替えて使っているような状況では
本アプリも関係はありますが本アプリを使わずともできることですので
アーキエイジのアップデートによりsystemcfgの変更があった場合はクライアントの終了時に
それが反映されますが
一度おかしいなと感じたらsystemcfgを削除してからクライアントの起動をしたほうが良いでしょう
これに関しては本アプリは関係ありません
